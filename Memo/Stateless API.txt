Stateless API: 서버가 클라이언트의 상태(state)를 저장하지 않는 API

특징
- 클라이언트의 모든 요청은 독립적이다.
- 각 요청에 필요한 모든 정보는 요청 자체에 담겨 있다.

Stateless API는 서버가 클라이언트의 상태(state)를 저장하지 않는 API를 말합니다. 클라이언트의 모든 요청은 독립적이며, 각 요청에 필요한 모든 정보는 요청 자체에 담겨 있습니다.

How to work?
서버는 요청을 받을 때마다 처음부터 모든 것을 처리한다. 
 예를 들어, 클라이언트가 로그인한 후 어떤 정보를 요청하면, 각 요청에 인증 토큰과 같은 정보가 포함되어야 한다. 서버는 이 토큰을 검증하고, 요청을 처리한 후, 요청과 관련된 어떤 상태 정보도 저장하지 않고 다음 요청을 기다린다.
